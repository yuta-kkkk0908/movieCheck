# Codex 運用設定

更新日: 2026-02-25

## 基本方針

1. 修正は一括ではなく、必要な箇所のみを最小差分で行う。
2. `TASKS.md` 内タスク参照は表記ゆれを許容して解釈する。
   - 例: `1`, `#1`, `タスク1`, `タスク１`, `Task 1`
3. 変更を伴う修正を実施した場合は、`CHANGELOG.md` に履歴を追記する。
4. 仕様に関する変更は、必ず `SPEC.md` と `TASKS.md` の両方を更新する。
5. 実装途中から再開する場合は、下記「再開用プロンプト」を使用する。

## タスク参照ルール

- 全角/半角数字を同一視する（`1` と `１`）。
- `タスク` / `task` / `#` などの接頭辞は省略可能として扱う。
- 曖昧な指定が複数候補に一致する場合は、`TASKS.md` の番号を優先し、必要なら確認する。

## 再開用プロンプト

以下をそのまま使えるテンプレートとして定義する。

```text
前回の続きから再開してください。
対象タスク: タスクX（または #X）
参照ファイル: TASKS.md / SPEC.md / CHANGELOG.md
再開前に以下を実施してください:
1) 現在の実装差分確認
2) 対象タスクの未完了項目の特定
3) 必要箇所のみ修正
4) 仕様変更があれば SPEC.md と TASKS.md 更新
5) 変更内容を CHANGELOG.md に追記
最後に、実施内容と未完了事項を報告してください。
```
